Help & Usage Guide

This page explains how to run the local controller and use the web UI to start and stop the SolSync copy-trading bot.

Quick Start

  1. Install dependencies in your Python environment:
    pip install -r requirements.txt
    or individually: pip install Flask websockets aiohttp solders
  2. Start the controller server: python frontend/python/control_server.py
  3. Open http://127.0.0.1:5000 in your browser.
  4. Fill in the form and click Start Bot. Use Stop Bot to terminate.

Form Fields

  • Trading Wallet Secret (base58): Your Solana wallet secret key encoded in Base58 (see “Key Concepts”). This stays on your machine and is only used by the local controller to sign transactions.
  • Wallet To Track (public key): The master wallet whose token balance changes the bot mirrors.
  • Fixed Buy Amount (SOL): Amount of SOL to spend per buy signal.
  • Sell Percent (%): Percentage of token balance to sell on a sell signal (e.g., 100).
  • Slippage (%) and Priority Fee (SOL): Trade execution parameters forwarded to the Pump/Raydium route.
  • Pool: Preferred route (Pump or Raydium). The bot rotates on retry.
  • RPC URLs: WebSocket and HTTP endpoints used to subscribe and submit transactions.

Logs & Status

  • The Status panel shows whether the bot process is running and its PID.
  • The Logs area shows timestamped controller messages and [BOT]-prefixed output from the bot.

Key Concepts

  • Base58 key: Solana keys are encoded with Base58 (an alphabet excluding easily-confused characters). Your “secret key (base58)” is a long Base58 string that decodes to your private key bytes. Keep it secret. Never share or commit it.
  • Public key: The Base58-encoded public address of a wallet (safe to share).
  • WebSocket subscribe: The bot subscribes to SPL Token program updates for the tracked wallet to detect balance changes quickly.
  • Trade execution: The bot requests a serialized transaction from Pump’s /trade-local API, signs it locally, and submits it via your configured HTTP RPC.

Security Tips

  • Use a dedicated hot wallet with limited funds.
  • Run the controller locally or behind a VPN/Zero Trust proxy if you expose it on the internet.
  • Never paste secrets into shared screens; rotate keys if leaked.
